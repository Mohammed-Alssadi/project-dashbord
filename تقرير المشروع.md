## 🔴 [خطر شديد #4] — `state` ثابت في OAuth يُمكّن هجمات CSRF

**الملفات:**
- `sallaAuthController.js` السطر 23: `&state=salla_secure_state`
- `zidAuthController.js` السطر 13: `&state=zid_secure_state`

**المشكلة:** الـ `state` في OAuth يجب أن يكون عشوائياً وفريداً لكل طلب لمنع هجمات CSRF (Cross-Site Request Forgery). استخدام قيمة ثابتة يجعل عملية المصادقة بالكامل عرضة للاختراق.

**الإصلاح الصحيح:**
```javascript
import crypto from 'crypto';
// أنشئ state عشوائي واحفظه في session مؤقت
const state = crypto.randomBytes(16).toString('hex');
// ثم تحقق منه في الـ callback قبل المعالجة
```

## 🔴 [خطر شديد #5] — Logout لا يمسح الكوكي بشكل صحيح

**الملف:** `backend/src/controllers/auth/authController.js`

```javascript
export const logout = (req, res) => {
  res.cookie('token', 'none', {
    expires: new Date(Date.now() + 10 * 1000), // ← ينتهي بعد 10 ثواني فقط!
    httpOnly: true
    // ← مفقود: secure: true, sameSite: 'lax'
  });
  res.json({ success: true, data: {} });
};
```

**المشاكل:**
1. الكوكي لا يُمسح فوراً — ينتهي فقط بعد 10 ثواني، وخلالها الجلسة لا تزال صالحة
2. الطريقة الصحيحة هي `res.clearCookie()` أو `maxAge: 0`
3. مفقود `secure: true` و `sameSite` — الكوكي الممسوح قد لا يطابق الكوكي المُنشأ أصلاً!
4. `logout` لا يتطلب مصادقة (`protect` غير مطبق عليه) — أي أحد يستطيع استدعاءه

**الإصلاح الصحيح:**
```javascript
export const logout = (req, res) => {
  res.clearCookie('token', {
    httpOnly: true,
    secure: true,
    sameSite: 'lax'
  });
  res.json({ success: true, message: 'تم تسجيل الخروج بنجاح' });
};
```

## 🟠 [خطر متوسط #7] — لا يوجد Rate Limiting

لا توجد أي حماية ضد:
- هجمات Brute Force على كل المسارات
- إغراق نقاط OAuth بطلبات متكررة
- استنزاف كوتة API لسلة وزد بطلبات مصطنعة

**الإصلاح:** أضف `express-rate-limit`:
```javascript
import rateLimit from 'express-rate-limit';
const limiter = rateLimit({ windowMs: 15 * 60 * 1000, max: 100 });
app.use('/api/', limiter);
```

## 🟠 [مشكلة] — لا يوجد Global Error Handler في Express

لا يوجد `app.use((err, req, res, next) => {...})` في `server.js`. أي خطأ غير متوقع سيُسقط الخادم أو يترك الطلب معلقاً بدون استجابة.

**الإصلاح:**
```javascript
// في نهاية server.js — بعد كل المسارات
app.use((err, req, res, next) => {
  console.error('Unhandled error:', err);
  res.status(err.status || 500).json({
    success: false,
    message: process.env.NODE_ENV === 'production' ? 'حدث خطأ في الخادم' : err.message
  });
});
```
