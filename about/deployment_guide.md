# الدليل النهائي لنشر منصة DashAI ككتلة واحدة (Single Subdomain Setup) على Hostinger

يغطي هذا الدليل خطوات نشر التطبيق بالكامل (الفرونت إند React والباك إند Node.js) مدمجين معاً على **نطاق فرعي واحد فقط (Subdomain)** (مثال: `dash.example.com`) من الصفر 100% وبأعلى كفاءة واحترافية.

---

## 🏗️ 1. المبدأ الهيكلي للدمج (Unified Architecture)

يقوم خادم **Express** في هذا النموذج بدورين متكاملين:
1. **مستقبل واجهة برمجة التطبيقات (API Gateway):** يستقبل طلبات الـ HTTP والـ OAuth على المسارات (`/auth`, `/stores`, `/health`).
2. **خادم ملفات الواجهة (React Static Server):** يقرأ ملفات واجهة React المضغوطة بمجلد `dist` ويخدمها استاتيكياً للمتصفح. في حال طلب المستخدم أي رابط فرعي (مثل `/dashboard/stores`)؛ يقوم السيرفر بتوجيهه لملف `index.html` تلقائياً ليعالجها الـ React Router دون إحداث خطأ 404.

---

## 🛠️ 2. المرحلة الأولى: تحضير الملفات محلياً (Local Preparation)

قبل بدء عملية الرفع إلى الاستضافة، يجب دمج مشروعي الفرونت إند والباك إند في جهازك المحلي كالتالي:

### 1) بناء واجهة React (Frontend Build):
* افتح التيرمنال في مجلد الـ `frontend` بجهازك المحلي وشغّل أمر البناء والضغط:
  ```bash
  npm run build
  ```
  * **النتيجة:** سيقوم React بإنتاج مجلد محسن ومضغوط بالكامل باسم **`dist`** يحتوي على ملف `index.html` ومجلد `assets` (ملفات الـ JS والـ CSS المدمجة).

### 2) دمج المجلدات (Folders Merging):
* قم بنسخ مجلد **`dist`** الناتج بالكامل من مجلد الفرونت إند، والصقه **داخل مجلد الباك إند (`backend/`)** بجانب ملف `server.js`.
* **هيكل ملفات الباك إند المحلي قبل الرفع يجب أن يبدو هكذا بدقة:**
  ```text
  backend/
  ├── dist/                <-- مجلد الفرونت إند المنسوخ
  │   ├── assets/          <-- ملفات الجافا سكربت والتصميم
  │   └── index.html       <-- الصفحة الرئيسية للواجهة
  ├── src/                 <-- الكود المصدري للباك إند (models, controllers, routes...)
  ├── database/            <-- الميجريشنز (migrations)
  ├── server.js            <-- ملف تشغيل خادم Express الرئيسي
  ├── package.json         <-- قائمة الحزم والسكربتات
  └── package-lock.json
  ```

### 3) ضغط الملفات للرفع:
* قم بضغط محتويات مجلد الباك إند (`backend`) بالكامل إلى ملف بصيغة **`.zip`** (مثال: `project-backend.zip`).
* ⚠️ **تحذير هام:** **لا تقم بضغط مجلد `node_modules`** نهائياً لتفادي كبر حجم الملف وسيقوم خادم Hostinger بتوليده تلقائياً.

---

## 🗄️ 3. المرحلة الثانية: تهيئة قاعدة بيانات MySQL في Hostinger

1. اذهب إلى لوحة تحكم Hostinger hPanel وافتح قسم **Databases (قواعد البيانات)** -> **MySQL Databases**.
2. قم بإنشاء قاعدة بيانات جديدة بملء الحقول:
   * **MySQL Database Name:** اكتب اسم قاعدة البيانات (مثال: `u12345_dashai_db`).
   * **MySQL Username:** اسم مستخدم قاعدة البيانات (مثال: `u12345_dashai_user`).
   * **Password:** كلمة مرور قوية لقاعدة البيانات.
3. اضغط على **Create**.
4. **تهيئة الجداول (Import Tables):**
   * بجانب قاعدة البيانات المنشأة حديثاً، اضغط على زر **Enter phpMyAdmin**.
   * من القائمة العلوية اضغط على **Import** (استيراد).
   * اختر ملف الـ SQL الخاص بالمشروع وهو **`backend/src/config/schema.sql`**، ثم اضغط على **Go** بالأسفل ليتم بناء جداول قاعدة البيانات والـ Indexes بالكامل فوراً.

---

## 💻 4. المرحلة الثالثة: إعداد خادم Node.js والرفع على النطاق الفرعي

1. في لوحة hPanel، ابحث عن قسم **Node.js** واضغط على **Create Application**.
2. قم بتهيئة النطاق الفرعي المخصص للمشروع (مثال: `dash.example.com`) بالخيارات التالية:
   * **Application Source Directory:** المجلد الذي سترفع فيه الملفات (تلقائياً هو مجلد النطاق الفرعي مثل `/public_html`).
   * **Domain/URL:** اختر النطاق الفرعي المخصص لك (مثال: `dash.example.com`).
   * **Application Startup File:** اكتب اسم ملف التشغيل الرئيسي للباك إند وهو: `server.js`.
   * **Node.js Version:** اختر أحدث نسخة مستقرة (v18 أو v20).
3. اضغط على **Create**.

### 📂 رفع وفك الضغط عن الملفات:
1. افتح **مدير الملفات (File Manager)** للنطاق الفرعي في Hostinger.
2. ارفع ملف الـ **`project-backend.zip`** الذي قمنا بضغطه محلياً سابقاً إلى مجلد النطاق الفرعي.
3. اضغط بزر الماوس الأيمن على الملف المرفوع واختر **Extract** (فك الضغط) لفك كافة محتويات المشروع هناك.

### ⚙️ إعداد متغيرات البيئة (`.env`):
في مدير ملفات السيرفر، أنشئ ملفاً جديداً باسم **`.env`** في جذر مجلد الباك إند (بجانب `server.js`) واكتب فيه المتغيرات الفعلية للإنتاج:

```env
# المنفذ الافتراضي لخادم Hostinger
PORT=3000

# بيانات قاعدة بيانات MySQL التي أنشأتها في Hostinger hPanel
DB_HOST=localhost
DB_USER=اسم_المستخدم_في_هوستينجر
DB_PASSWORD=كلمة_مرور_قاعدة_البيانات
DB_NAME=اسم_قاعدة_البيانات_في_هوستينجر

# رابط النطاق الفرعي المخصص لك (الموقع الفعلي)
FRONTEND_URL=https://dash.example.com

# مفاتيح الربط لمنصة سلة (OAuth)
SALLA_CLIENT_ID=معرف_تطبيق_سلة_الخاص_بك
SALLA_CLIENT_SECRET=مفتاح_سلة_السري
SALLA_REDIRECT_URI=https://dash.example.com/auth/salla/callback

# مفاتيح الربط لمنصة زد (OAuth)
ZID_CLIENT_ID=معرف_تطبيق_زد_الخاص_بك
ZID_CLIENT_SECRET=مفتاح_زد_السري
ZID_REDIRECT_URI=https://dash.example.com/auth/zid/callback
```

> [!IMPORTANT]
> تأكد من تعديل الـ **Redirect URI** في لوحة مطوري سلة وزد ليتطابق مع الرابط الجديد لنطاقك الفرعي الفعلي: `https://dash.example.com/auth/salla/callback` و `https://dash.example.com/auth/zid/callback`.

---

## 🚀 5. المرحلة الرابعة: تشغيل التطبيق (Run and Go)

1. ارجع إلى صفحة تطبيق Node.js في لوحة Hostinger hPanel.
2. اضغط على زر **Run npm install** لتثبيت حزم الـ node في السيرفر تلقائياً.
3. اضغط على زر **Start Application** (بدء تشغيل التطبيق).
4. افتح المتصفح واذهب للنطاق الفرعي الخاص بك (`https://dash.example.com`) وستجد المنصة كاملة تعمل بنجاح فوري ككتلة واحدة مستقرة وسريعة التحميل!
